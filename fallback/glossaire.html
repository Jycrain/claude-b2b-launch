01

Intelligence artificielle & produit Claude

Le vocabulaire du produit qu'on lance et de la technologie qui le sous-tend.

IA générative
Intelligence artificielle capable de produire du contenu nouveau (texte, code, images) plutôt que de seulement classer ou prédire. C'est la catégorie de marché du projet.
LLM — Large Language Model / grand modèle de langage
Modèle d'IA entraîné sur d'immenses corpus de texte pour comprendre et générer du langage. Claude, ChatGPT et Gemini sont des LLM.
Anthropic
L'entreprise qui édite Claude, fondée en 2021, centrée sur la sécurité de l'IA. C'est « l'entreprise existante » dont on simule le lancement d'un nouveau produit.
Opus / Sonnet / Haiku
Les trois tailles de modèles Claude : Opus (le plus puissant, raisonnement complexe), Sonnet (équilibre qualité/coût, le modèle par défaut), Haiku (le plus rapide et économique, gros volumes).
Token
Unité de base que lit/écrit un LLM (≈ ¾ d'un mot). La facturation et la longueur des échanges se comptent en tokens.
Fenêtre de contexte
Quantité de texte (en tokens) que le modèle peut « avoir en tête » en une seule fois. Claude en propose une très large (~1 million de tokens) : ingérer un contrat ou une base de code entière d'un coup.
Claude Code
L'outil de programmation agentique de Claude : il lit un dépôt de code, écrit, teste et ouvre des « pull requests » de façon autonome — pas seulement des suggestions. → dans le projet : c'est aussi l'outil qui a servi à coder cette présentation.
Agentique / Agent IA
Une IA qui exécute des tâches en plusieurs étapes de façon autonome (agir), au lieu de seulement répondre. Tendance forte du marché entreprise.
MCP — Model Context Protocol
Standard ouvert qui permet de brancher un LLM aux outils internes d'une entreprise (drives, tickets, bases de données) sans développement sur-mesure.
Constitutional AI
Méthode d'entraînement d'Anthropic où le modèle suit une « constitution » de principes explicites pour être utile, honnête et inoffensif — la fiabilité par conception. C'est le différenciateur clé de Claude.
Hallucination
Quand un LLM produit une information fausse mais présentée comme vraie. Un risque que les décideurs (DSI, juridique) redoutent.
Prompt
La consigne/question envoyée au modèle. La qualité du prompt conditionne la qualité de la réponse.
02

Sécurité, conformité & souveraineté

Le cœur de l'argumentaire B2B : ce qui « débloque les comités sécurité ».

SOC 2 — Type I & Type II
Certification d'audit américaine sur la sécurité des données. Type I = les contrôles existent à un instant T ; Type II = ils fonctionnent dans la durée (plus exigeant).
ISO 27001
Norme internationale de référence pour le management de la sécurité de l'information.
ISO/IEC 42001
Première norme de management de l'IA (2023) — rare sur le marché. Atout différenciant de Claude pour la gouvernance.
HIPAA
Loi américaine protégeant les données de santé. « HIPAA-ready » = utilisable dans la santé sous contrat (BAA).
FedRAMP
Programme de certification de sécurité cloud du secteur public américain (« FedRAMP High » = niveau élevé).
RGPD — GDPR en anglais
Règlement européen sur la protection des données personnelles. S'applique dès qu'un usage IA traite des données de personnes.
AI Act
Règlement européen sur l'IA. Échéance clé : pleine applicabilité le 2 août 2026, sanctions jusqu'à 7 % du chiffre d'affaires mondial. Moteur de l'urgence côté décideurs.
GPAI — General-Purpose AI
« Modèle d'IA à usage général » au sens de l'AI Act (comme Claude) : obligations de documentation et de transparence spécifiques.
CNIL
L'autorité française de protection des données. A prononcé ~487 M€ de sanctions en 2025 — argument de risque dans le projet.
DPA — Data Processing Agreement
Contrat encadrant le traitement des données personnelles entre un client et son fournisseur, exigé par le RGPD.
Zero Data Retention (ZDR) / non-entraînement
ZDR : les données ne sont pas conservées après la réponse. Non-entraînement : les données client ne servent pas à entraîner le modèle. Deux garanties contractuelles décisives.
SSO — Single Sign-On
Authentification unique : se connecter à Claude avec les identifiants d'entreprise (Okta, Azure AD) déjà existants.
SCIM
Standard d'approvisionnement automatique des comptes : créer/supprimer les accès des collaborateurs automatiquement depuis l'annuaire RH/IT.
RBAC — Role-Based Access Control
Gestion des droits par rôle : chaque profil n'accède qu'à ce qui le concerne.
SIEM
Outil de supervision de sécurité (Splunk, Datadog…) qui centralise les journaux. Claude peut y exporter ses logs d'audit.
Allowlisting IP
Restreindre l'accès à une liste d'adresses IP autorisées (ex. le réseau de l'entreprise).
Shadow IT
Outils utilisés par les salariés hors contrôle de la DSI (ex. un LLM grand public). Un risque que l'offre Enterprise vise à « gouverner ».
Souveraineté / résidence des données
Résidence : où sont physiquement stockées les données (ex. UE). Souveraineté : garder la maîtrise juridique et technique de ses données.
Cloud Act
Loi US permettant aux autorités américaines d'accéder à des données détenues par des entreprises US, même hébergées hors des États-Unis. Crainte fréquente en Europe.
Trust Center / Trust Portal
Espace en ligne où un éditeur publie ses certifications et documents de sécurité (souvent sous accord de confidentialité, « NDA »).
03

SEO — référencement naturel

Comment une page se positionne « gratuitement » dans Google.

SEO — Search Engine Optimization
L'ensemble des techniques pour qu'une page apparaisse haut dans les résultats de recherche naturels (non payants).
Balises Hn (H1 → H6)
Les niveaux de titres d'une page web, comme un plan hiérarchique : H1 = titre principal (un seul par page), H2 = grandes sections, H3 = sous-sections… jusqu'à H6. Ils structurent le contenu pour le lecteur ET pour Google. → dans le projet : on a cartographié les Hn de la page Claude Enterprise pour repérer les manques.
Balise Title
Le titre de l'onglet/du résultat Google. Très important pour le clic et le référencement.
Meta description
Le court texte descriptif affiché sous le titre dans les résultats Google. N'influence pas le classement mais le taux de clic.
SERP — Search Engine Results Page
La page de résultats de Google pour une requête donnée.
Mot-clé & intention (intent)
Mot-clé : la requête tapée. Intention : ce que cherche vraiment l'internaute (s'informer, comparer, acheter). On cible des mots-clés à intention commerciale.
Maillage interne
Les liens entre les pages d'un même site. Un bon maillage renforce l'autorité des pages importantes.
TOFU / MOFU / BOFU
Étapes de l'entonnoir de contenu : TOFU (haut, découverte), MOFU (milieu, évaluation), BOFU (bas, décision — ex. pages comparatives « Claude vs ChatGPT »).
Trafic organique
Les visites venant des résultats de recherche naturels (par opposition au trafic payant).
Redirection 301 / canonicalisation
301 : redirection permanente d'une URL vers une autre. Canonicalisation : désigner l'URL « officielle » d'un contenu pour éviter les doublons. Trop de 301 « diluent » le référencement.
Featured snippet / People Also Ask
Encadrés de réponse mis en avant par Google (souvent à partir de contenus en FAQ). On peut les viser avec des titres en forme de questions.
E-E-A-T
Critères de qualité Google : Experience, Expertise, Authoritativeness, Trustworthiness. Les preuves chiffrées et témoignages les renforcent.
04

Performance web — PageSpeed / Lighthouse

Mesurer la vitesse et la qualité technique d'un site.

PageSpeed Insights
Outil gratuit de Google qui note une page web et donne des recommandations de performance.
Lighthouse
Le moteur d'audit (intégré à Chrome) derrière PageSpeed. Donne 4 scores sur 100 : Performance, Accessibilité, Bonnes pratiques, SEO. → dans le projet : on a mesuré Claude.com vs OpenAI.com avec Lighthouse, et notre propre app.
Core Web Vitals
Les indicateurs de Google sur l'expérience de chargement réellement ressentie : LCP, CLS, INP.
LCP — Largest Contentful Paint
Temps d'affichage du plus gros élément visible (image/titre). Mesure la vitesse de chargement perçue. Bon < 2,5 s.
FCP — First Contentful Paint
Temps avant l'apparition du premier contenu à l'écran.
CLS — Cumulative Layout Shift
Mesure la stabilité visuelle : à quel point la page « saute » pendant le chargement. Plus c'est proche de 0, mieux c'est. → dans le projet : Claude ~0 (excellent) vs ChatGPT 0,05–0,08.
INP / FID
INP (Interaction to Next Paint, qui remplace l'ancien FID) mesure la réactivité de la page aux clics.
TBT — Total Blocking Time
Temps pendant lequel la page est « figée » (le navigateur traite du JavaScript) et ne répond pas. Proxy de l'INP en labo.
TTFB — Time To First Byte
Délai avant de recevoir le premier octet du serveur. Mesure la réactivité serveur/réseau. → dans le projet : Claude ~40 ms (excellent, via CDN).
Lab vs field (CrUX)
Lab : mesure en conditions de test contrôlées. Field / CrUX : données réelles d'utilisateurs Chrome. Un score lab peut différer du terrain (d'où le « à confirmer » sur la perf concurrente).
CDN — Content Delivery Network
Réseau de serveurs répartis dans le monde qui servent le contenu depuis le point le plus proche → site plus rapide.
Bundle / code-splitting / lazy loading
Bundle : le fichier JavaScript de l'app. Code-splitting : le découper en morceaux. Lazy loading : ne charger un morceau qu'au besoin → démarrage plus rapide. → dans le projet : la section avec graphiques est chargée à part.
gzip / Brotli (compression)
Compression des fichiers envoyés par le serveur pour réduire leur poids et accélérer le chargement.
woff2
Format de police web compressé moderne. Les polices du projet (Fraunces, Inter) sont en woff2 et embarquées localement (offline).
05

UX & architecture de l'information

L'organisation et l'expérience de navigation du site.

UX / UI
UX (User Experience) = l'expérience d'usage globale. UI (User Interface) = l'apparence visuelle de l'interface.
Arborescence / sitemap
Le plan hiérarchique des pages d'un site (menus et sous-menus). → dans le projet : on montre l'arborescence desktop & mobile et où s'insère le produit Enterprise.
Funnel (entonnoir de conversion)
Le parcours du visiteur étape par étape (découverte → évaluation → achat). À chaque étape, on perd du monde — d'où l'image d'entonnoir.
Menu hamburger
L'icône à trois barres ☰ qui replie le menu sur mobile. Différence classique desktop/mobile.
CTA — Call To Action
Bouton/lien incitant à l'action (« Contact sales », « Try Claude »). Sa clarté influence la conversion.
Scrollytelling
Technique de présentation où le récit se déroule au fil du défilement (scroll), avec animations. C'est le format de cette présentation (au lieu d'un PowerPoint).
06

Marketing B2B & vente

Le vocabulaire du plan d'action et des personas.

B2B / B2C
B2B (Business to Business) : vendre à des entreprises. B2C : vendre à des particuliers. Notre lancement est B2B.
Persona
Portrait-type d'un client cible, incarné (nom, âge, métier, motivations, freins). Sert à adapter le message. → dans le projet : 3 personas — Sophie (DSI), Karim (CTO), Élodie (juridique).
CSP — Catégorie socioprofessionnelle
Classification du métier/statut social (ex. « cadre dirigeant », « CSP+ »). Décrit le profil d'un persona.
ABM — Account-Based Marketing
Marketing ciblé compte par compte : on identifie nominativement quelques grands comptes et on leur adresse un contenu sur-mesure. Cœur de l'Action 01.
Lead / MQL / SQL
Lead : contact intéressé. MQL (Marketing Qualified Lead) : lead qualifié par le marketing. SQL (Sales Qualified Lead) : lead jugé mûr pour la vente.
Retargeting / display / affiliation / influence
Leviers publicitaires : retargeting (recibler les visiteurs), display (bannières), affiliation (commission sur vente apportée), influence (créateurs/experts).
Land-and-expand
Stratégie : « atterrir » avec un petit déploiement (pilote), puis étendre progressivement à toute l'organisation. Mesuré par le NRR.
Pipeline
L'ensemble des opportunités de vente en cours, valorisées en euros. Indicateur de santé commerciale.
Cycle de vente
Le temps entre le premier contact et la signature. En B2B grands comptes, il est long (revues sécurité, achats…).
ESN — Entreprise de Services du Numérique
Sociétés d'intégration/conseil IT (Sfeir, Capgemini, Sopra…). Partenaires pour déployer Claude chez les clients (Action 03).
Comportements digitaux
Les habitudes en ligne d'un persona (réseaux suivis, contenus consommés). Sert à choisir les canaux.
Points de friction
Les obstacles/objections qui freinent l'achat (ici : risque de fuite de données, coût, conformité).
Proposition de valeur
La promesse centrale : pourquoi choisir ce produit plutôt qu'un autre, exprimée en bénéfices pour le client.
07

KPI & finance

Les indicateurs qui mesurent le succès du plan.

KPI — Key Performance Indicator
Indicateur clé de performance : une mesure chiffrée d'un objectif. → dans le projet : exactement 6 KPI reliés aux 3 actions.
Baseline
La valeur de départ (le « point zéro ») à partir de laquelle on mesure la progression d'un KPI.
CAC — Coût d'Acquisition Client
Combien coûte (marketing + vente) l'acquisition d'un nouveau client.
ACV — Annual Contract Value
Valeur annuelle moyenne d'un contrat. Un bon ratio cible : CAC < ⅓ de l'ACV.
ROI / ROAS
ROI : retour sur investissement global. ROAS (Return On Ad Spend) : chiffre d'affaires généré par euro de pub.
NRR — Net Revenue Retention
% de revenu conservé + développé sur les clients existants d'une année sur l'autre. > 100 % = la base croît seule (land-and-expand réussi).
Taux d'activation
% d'utilisateurs qui adoptent réellement l'outil après inscription (ex. comptes actifs à 2,5 semaines).
Time-to-contract
Délai entre la qualification d'un prospect (SQL) et la signature. Plus il est court, plus la machine commerciale est efficace.
Run-rate
Revenu annualisé extrapolé à partir d'une période récente (ex. « 47 Md$ de run-rate » = rythme annuel projeté).
CAGR
Taux de croissance annuel moyen sur plusieurs années (Compound Annual Growth Rate).
TCO — Total Cost of Ownership
Coût total de possession : prix du siège + usage + intégration. Ce que regarde vraiment un acheteur.

Outils utilisés

Les outils du syllabus mobilisés pour l'analyse (un rôle unique chacun).

SimilarWeb
Outil d'estimation du trafic et de l'audience d'un site (volumes, géographie, sources). Sert à comparer claude.ai aux concurrents.
Google Search
Utilisé pour analyser la visibilité de marque, les mots-clés et la concurrence dans les résultats (SERP).
GlooMaps
Outil de création de plans de site / arborescences (sitemaps visuels).
Web Developer (extension)
Extension navigateur pour inspecter une page : balises, métadonnées, responsive, accessibilité.
SEO Review Tools — Headings Checker
Outil qui extrait la hiérarchie des titres Hn (H1→H6) d'une page.
Canva
Outil de création graphique pour les visuels de campagne et la direction artistique.
Excel / Google Sheets
Tableurs pour le scoring personas, le tableau de bord des KPI, le budget et les ratios (CAC/ROAS).
Looker Studio
Outil Google de tableaux de bord pour piloter les KPI en temps réel.
09

Stack technique & déploiement

Comment l'application a été construite et mise en ligne.

Vite
Outil moderne qui compile et sert l'application web (rapide, avec rechargement à chaud en développement).
React
Bibliothèque JavaScript pour construire des interfaces en composants réutilisables. Base de l'app.
TypeScript
JavaScript avec typage : détecte des erreurs avant l'exécution → code plus fiable.
Framer Motion
Bibliothèque d'animations React (apparitions au scroll, transitions douces de la présentation).
Recharts
Bibliothèque de graphiques React (le radar concurrentiel, les visualisations de données).
Offline-safe
L'app fonctionne sans connexion : aucune requête réseau au chargement (polices et données embarquées). Crucial pour une soutenance en huis clos.
Build / dist
Build : l'étape qui transforme le code source en fichiers optimisés. dist : le dossier de sortie statique prêt à servir.
base path
Le préfixe d'URL sous lequel l'app est servie (ex. /claude-b2b-launch/ sur GitHub Pages, / en local).
Git / GitHub
Git : système de gestion de versions du code. GitHub : plateforme d'hébergement de ces dépôts.
GitHub Pages
Service d'hébergement de sites statiques gratuit de GitHub. → dans le projet : héberge la démo en ligne.
CI/CD — GitHub Actions
CI/CD : automatisation de l'intégration et du déploiement. GitHub Actions : l'outil qui, à chaque envoi de code, rebuild et redéploie la démo automatiquement.
Playwright
Outil qui pilote un navigateur de façon automatisée : captures d'écran, audits Lighthouse, tests — utilisé pour mesurer et exporter le PDF.
